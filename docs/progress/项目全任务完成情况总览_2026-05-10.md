## 0. 当前总判断

项目已经从“Trellis2 递归工程管线”进一步收束为可投稿方向的保守主线：

> 在冻结 3D generator 的原生 sparse support / sparse latent state 上执行 finite-depth recursive grammar；grammar 负责 handles、rule proposal、递归状态和 projection admissibility，Trellis2 负责 sparse state 接口、decode/re-encode、局部 naturalization 可能性，以及 selected meshes 的 texture/PBR export compatibility。

当前可以相对稳妥主张：

- **方法定位**：generation-model-native recursive grammar / recursive language over sparse 3D latents，而不是 projection-first mesh cleanup。
- **结构证据**：per-depth projection / admissible state selection 可以在若干 conservative case 中抑制 fragment 传播；occupancy/support connectedness 是主指标之一。
- **资产化证据**：selected projected meshes 可以进入 frozen Trellis2 texture/PBR export path；这证明 export compatibility，不证明 topology clean。
- **计算诊断**：summary-only runtime/token-growth profiling 已完成，支持 token/mesh scaling 与 decode-time 诊断，但不是视觉质量或拓扑证据。

当前不能主张：

- sparse-latent grammar 已严格优于 mesh-space procedural recursion + Trellis2 texture export。
- 所有 textured GLB 都 topology-clean 或 watertight。
- DLA / hard-DLA / physical crystal growth 已解决。
- Flow/SDE 或 masked naturalization 已被证明能稳定改善表面且不造成 topology drift。
- Infinite recursion、effective resolution 或 one-shot Trellis2 对比优势已闭环。

   摘要、contribution、claim map 中涉及 root/tip/branch preservation、recursive structure 的量化 claim 仍需证据。
