## 0. 当前主故事框架

目前最稳的论文主线建议固定为：

> **Projection-Stabilized Recursive Sparse-Latent Grammar (PS-RSLG)**：把递归 3D asset growth 定义为 frozen native 3D generative representation 中的反复状态转移。Grammar 负责 sparse support、attachment、frontier、space competition 和 transform-copy；Trellis2/flow/SDE 负责局部自然化、稀疏 latent 表达和可选 texture/PBR export；关键算法设计是把 decode -> projection/pruning -> re-encode 放进每一层递归，而不是只在最终深度做一次 cleanup。

这条故事的优势：

- 可以自然解释为什么要用 Trellis2：不是只为了更好看，而是因为 O-Voxel/SLAT 是可被 grammar 修改、投影、重编码的稀疏 3D 状态。
- 可以覆盖传统 procedural families：IFS、L-system、space colonization、DLA/frontier、shape grammar、symmetry/crystal transform-copy 都可作为 rule semantics 的特例。
- 可以把已经跑出来的正面结果放到一个统一 claim 下：`compete` 稳定、per-depth projection 抑制碎片传播、mesh-first 比图像草图更可靠、selected cases 可导出 textured GLB。

这条故事的风险：

- 目前证明和公式已经比 1540 文档强很多，但仍偏“覆盖性/稳定性 sketch”，还没有达到完全严密的理论论文级别。
- 实验支撑不均衡：tree/vine/root 线最稳；non-tree/porous/crystal/Escher 是候选和压力测试，不能过早作为主 claim。
- 视觉仍是最大风险：已有 mesh/GLB，但 PBR/texture 质量和类别覆盖不足，头图还不成立。

目前 formal doc 里的核心对象已经扩展为：

```tex
S_d=(C_d,F_d,U_d,A_d,B_d,M_d,H_d)
```

其中 `C_d` 是 sparse O-Voxel/SLAT support，`F_d` 是 features，`U_d` 是 decoded mesh/occupancy，`A_d` 是 anchor/frontier/tip graph，`B_d` 是 occupancy/competition buffer，`M_d` 是 mask/material/projection metadata，`H_d` 是 history/cache。

Grammar 定义为：

```tex
G=(\Sigma,\mathcal{T},R,I,P,N_\theta,\Pi,\mathrm{Caches})
```

其中 `R` 覆盖 typed rewriting / transform-copy / frontier growth / contextual replacement，`N_\theta` 是 Trellis2 flow/SDE/texture naturalization，`\Pi` 是 projection/competition/constraint family。

当前更强的递归公式：

```tex
S_{d+1}
=
\mathcal{C}_{d+1}\circ E\circ P_\lambda\circ
N_{\theta,\Omega}^{\eta\rightarrow 0}\circ
\Theta_\Pi\circ
\operatorname{Merge}_B\circ
\operatorname{Prop}_r(S_d)
```

已经把生成模型采样过程纳入 grammar 内部，而不是作为后处理。

### 已覆盖的传统系统

- IFS：contractive transform-copy + identity sampler + disabled projection。
- L-system：symbol rewriting + turtle/frame interpretation -> typed anchor rewriting。
- Space colonization：attractor-conditioned frontier proposal + occupancy exclusion。
- DLA/frontier：stochastic frontier attachment + exclusion buffer。
- Shape grammar：contextual split/extrude/replace over frames and masks。
- Symmetry/crystal：group action closure + approximately equivariant projection/sampler。

### 仍未完成 / 仍不够发表级

- 公式虽然已经能 cover 传统方法，但论文正文里还需要更“图形学式”的组织：定义、规则语法、操作语义、命题、证明草图、算法框、对应实验 claim。
- Symmetry/crystal 的等变性还只有 framework 和 proxy 实验，没有足够强的视觉和 metric。
- Infinite recursion 只有 LOD/cache proxy，不能作为主贡献，只能作为 extension 或 future direction。
- 目前方法总图是 raster draft，信息清晰但不够最终论文图。需要重新画成 vector 或高分辨率图，并嵌入真实 mesh/GLB case。

### 下一步

1. 把 formal doc 压缩成论文 Method 3-4 页版本：先定义 grammar，再给 algorithm box，再给 coverage proposition。
2. 为 `IFS/L-system/space-colonization/DLA/symmetry` 做一张 rule taxonomy 图，显示它们如何落入统一语法。

### 已明确的 metric 框架

## 8. 严格图形学视角下的结论

现在这篇工作的研究方向是有潜力的，但不能把它写成“我们拼了一个 Trellis2 递归工程 pipeline”。必须把核心贡献压到三点：

1. **语法表达力**：一个 typed sparse-latent grammar，形式上 subsume 经典递归/procedural families。
2. **递归稳定性**：per-depth projection/re-encode 是递归算子的一部分，数值上抑制 component fragmentation，算法上防止误差跨深度传播。
3. **生成式资产化**：frozen 3D generator 不替代 grammar，而是作为 sparse representation、local prior 和 appearance export，使 classical recursive structures 变成可渲染 mesh/GLB assets。

如果这三点都用实验支撑住，论文故事成立。  
如果最后只有几张好看的结果图但没有同协议 baseline/metric，容易被 reviewer 看成 engineering demo。  
如果只有公式但 PBR/mesh 视觉不够好，SIGGRAPH Asia 也很危险，因为任务是 visual computing。

当前最高优先级：

1. 把所有候选图转为 mesh/GLB/Blender 渲染，matplotlib 只当诊断。
2. 固化 space competition 主实验。
3. 提升 PBR/texture 质量，先凑齐四类可用头图资产。
4. 把 formal grammar 压进论文 Method，绑定 projection ablation 和 coverage taxonomy。
