## 2. 需要立刻尝试的算法路线

### 2.1 Bridge-aware projection

把投影从“保留最大连通分量”扩展为：

1. 识别主分量和候选碎片；
2. 根据距离、法向、语义/材质、父子 trace、grammar anchor 选择可桥接碎片；
3. 在 sparse occupancy 或 voxelized mesh 上补桥；
4. 对补桥后结果做 decode/re-encode 或局部平滑；
5. 再用 occupancy LCR、component count、视觉渲染共同筛选。

### 2.2 Connected sparse proposal

DLA 和 radial 目前的问题是每个 frontier / transform copy 像独立碎片。后续 rule proposal 必须带硬约束：

- 每个新增支持必须有到主支持的连接路径；
- 每个 proposal 的边界接触面积/邻接 voxel 数低于阈值则拒绝；
- 对晶体/径向复制，先生成连接骨架/母体，再复制晶面或局部块，而不是复制完整孤立 patch。

### 2.3 Cache fusion

cache 不应只是节约计算，而应作为“连接良好的 motif reuse”：

- motif cache 存的是连通 motif + attachment ports；
- transform cache 复制 motif 时必须对齐 port；
- LOD cache 在小尺度只保留 connected hull / simplified proxy；
- sliding-window cache 只让局部窗口高细节，窗口外保留连通 proxy。

### 2.4 Sampler/SDE under hard masks

生成模型采样不能全局重洗拓扑。只允许在硬连通 mask 内做局部自然化：

- mask 只覆盖新生长区域和桥接区域；
- anchor / bridge / main support 设为 hard clamp；
- 多样本采样后用连通性指标选样；
- 纹理/PBR export 放在结构筛选之后。

### 2.5 DLA / crystal 特化

铋晶体或 pyrite 这条线如果做通会很强，但不能用现在的 bottle / chunks 结果。可尝试：

- 先构造连通 crystal core 或 lattice skeleton；
- 只在 core 表面长晶面，不允许独立漂浮碎块；
- 对 `radial4` 使用 group-orbit closure + bridge-aware merge；
- 指标加入 symmetry error、component count、largest ratio、cavity/coverage、Blender visual QA。
