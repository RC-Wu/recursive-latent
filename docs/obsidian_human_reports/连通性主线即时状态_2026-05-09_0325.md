# 连通性主线即时状态 2026-05-09 03:25

## 当前判断

纹理和 PBR 质量已经可以作为当前研究迭代的可用基础，但 DLA、晶体、scifi、部分 root/leaf 结构的碎块化仍然是论文级结果的最大风险。现在主线从“头图/纹理继续打磨”转为“把连通性做成方法论不变量”。

## 已确认的负结果

第一轮 DLA/晶体远端实验没有产生有效候选，因为 `recursive_growth_mesh_metrics.py` 没有同步进远端 Python path，所有候选在 raw decode 后计算 metrics 时失败。已修复部署。

修复后的最小 smoke 能完整跑完，但 DLA `fork_side_attach + sparse_close_bridge` 是负结果：raw occupancy 已经接近连通（4 components, LCR 0.995），sparse close decode 后碎片反而增加，mesh projection 后 LCR 降到 0.787。这说明 naive sparse closing/post mesh repair 不足以解决问题。

本地传统 mesh repair 支线也显示：只有 radial4 类能被 voxel close 安全改善；DLA、scifi、leaf、bismuth 不能靠普通 hole fill/largest component/voxel close 宣称修好。

## 当前正在跑

远端 GPU 4/5/6/7 正在跑 `connectivity_first_dla_crystal_20260509_bridgefix_grid`，比较 raw bridge、mesh bridge、sparse close 三类策略，并已修正“大组件也必须桥接回主结构”的 projection bug。

## 新策略

同时启动 connected scaffold 路线：直接在 grammar/root 层生成保证 connected support 的 DLA/coral/bismuth/crystal/root scaffold，然后用 Trellis2 做 texture/PBR 或轻度自然化。这比在碎片 mesh 后面补洞更符合现在的方法论需求，也更可能得到论文可用的 DLA/晶体结果。

## 方法论影响

PS-RSLG 需要把 connected admissible state、projection operator、bridge/cache/naturalization loop 写成核心，而不是工程修补。实验上要同时报告正结果和负结果：普通后处理失败，connected-support grammar/projection 才是必要设计。
