# 即时进展：DLA 连通性 Smoke 与图形风格修复

## 结论摘要

今天中午的窄矩阵 smoke 基本确认：**DLA / 碎块类 case 不能继续靠 post-hoc bridge/cache 当主线正例**。hard DLA 即使做 sparse close bridge，occupancy 仍然是多组件；volumetric DLA 的正向信号来自生成时就保持 connected support，而不是后修。

当前主线应收束为：

> grammar-native connected scaffold -> per-depth projection -> selected Trellis2 textured GLB / PBR export。

这和之前用户指出的“碎块完全不能接受”一致：后修 bridge 可以作为负例和动机，但不能作为可用 asset 的解决方案。

## 论文含义

可以写：

- post-hoc bridge/cache 不是 DLA 碎块问题的可靠解；
- occupancy-primary metric 必须和 raw mesh face diagnostics 分开；
- DLA/coral/crystal 类正例要通过语法层 parent anchor、root reachability、face contact、per-depth projection 来保证 connected support；
- bridge/cache 的负结果支持 PS-RSLG 中 connected-state invariant 的必要性。

不能写：

- DLA 已解决；
- sparse close bridge 解决了碎块；
- face component 变成 1 就代表 asset topology clean；
- volumetric DLA 等价真实 DLA 物理过程。

## Blender 视检补充

固定 Blender/Cycles mesh render 后，结论更清楚：

- hard DLA raw：肉眼可见多块漂浮，完全不能用；
- hard DLA sparse bridge：出现长杆假桥，视觉上就是后处理缝合；
- volumetric DLA raw：当前四个里最像可用 connected scaffold，可作为 supplement stress case，但不要当头图；
- volumetric DLA sparse bridge：face 指标变好，但视觉上有射线状长桥，occupancy 也退化。

这说明后续非树 / DLA / coral 正线必须是生成规则层面的 parent anchor / root path / face contact，而不是继续做更强的后修。
